- Убраны отдельные UI-кнопки для пересчета ротации/наклона; это включено в основной экспортный пайплайн.

## What Was Changed

### 1) DB export model: one table `StarSystems`
Файл: `src/main/java/org/example/starforge/core/io/MySqlPerSystemTableExporter.java`

- Добавлен target table:
  - `TARGET_TABLE = "StarSystems"`
- `exportOne(...)` теперь принимает:
  - `int starSystemId` вместо имени таблицы.
- Перед вставкой выполняется очистка только по системе:
  - `DELETE FROM StarSystems WHERE StarSystemID=? AND ObjectType IN (1,2,3)`
- `INSERT` теперь включает `StarSystemID` в список колонок.
- `updateRotationInclination(...)` и `updateRotationSpeed(...)` обновляют строки по:
  - `StarSystemID + ObjectInternalID + ObjectType IN (2,3)`
- Для corrections mode:
  - `listStarSystemIds(...)` возвращает `SELECT DISTINCT StarSystemID ...`
  - `applyCloseMoonCorrection(...)` работает в рамках конкретного `StarSystemID`.
- Добавлена проверка наличия колонки `StarSystemID` в `StarSystems`.

### 2) UI pipeline cleanup
Файл: `src/main/java/org/example/starforge/gui/App.java`

- Основной экспорт продолжает считать систему и вызывать:
  - `exporter.exportOne(c, starSystemId, systemSeed, sr, sys);`
- Удалены кнопки:
  - `Update RotationInclination → MySQL`
  - `Update RotationSpeed → MySQL`
- Удалены их обработчики, фоновые задачи и связанная блокировка UI.
- В тексте подтверждения экспорта добавлено явное сообщение, что:
  - rotation speed и rotation inclination обновляются в процессе основного экспорта.

## Current Behavior

  - одновременно записывает орбитальные и ротационные поля.
- `Corrections Mode` обходит уникальные `StarSystemID` в `StarSystems`.
